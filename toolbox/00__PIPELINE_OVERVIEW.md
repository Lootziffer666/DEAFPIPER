# Vibe Coding Pipeline (Teil 1) – Übersicht (vorläufig)

Dieses Bundle bündelt die vorhandenen Inhalte in eine durchgehende Pipeline von **Idee → Research → PRD → UX/UI → Build → QA/Security/Compliance → Marketing/GTM → Measurement/Iteration**.

## Pipeline-Stufen (high-level)

1. **Prompting Interface & Standards**
 Ziel: Ein einheitliches Eingabe-/Ausgabeformat (weniger Reibungsverlust, reproduzierbare Ergebnisse).

2. **Research & Dokumente**
 Ziel: Fakten, Quellen, Requirements, Wettbewerbs-/Markt-Inputs, Dokument-Extraktion.

3. **Idee → Strategie → PRD**
 Ziel: Problem-Framing, Zielgruppe, MVP-Scope, Requirements, Success Metrics, Roadmap.

4. **UX/UI & Flows**
 Ziel: User Journey, IA/Flows, UX Writing, Design System Starter, Diagramme.

5. **Build / Vibe Coding**
 Ziel: Implementierung mit klaren Constraints, vollständigem Code, Tests, Run Instructions.

6. **QA / Security / Compliance**
 Ziel: Tests, Threat Model, Hardening, Legal/Claims/DSGVO-Checklisten.

7. **Marketing / GTM / Content**
 Ziel: Positionierung, Messaging, Testplan, Creatives, Measurement.

8. **Execution / Ops / Analytics**
 Ziel: 4-Wochen-Pläne, KPI/Dashboards, Reviews, Iteration.
